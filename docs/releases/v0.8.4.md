# MABW v0.8.4 — Safe Parallelism Foundations

MABW v0.8.4 adds the foundations for parallel work that does not contaminate
hash-sensitive workflow artifacts.

The release is intentionally narrow. It does not claim faster end-to-end
briefing, better prose, better factual support, or model-performance gains.
It makes unordered work safer to run in parallel by forcing deterministic joins,
keeping write transactions single-owner, and recording provenance only when it is
explicitly supplied.

## What's New

- **Deterministic source provider join.** Source provider results now pass
  through a stable join layer. Provider priority and stable batch indexes decide
  duplicate winners, not completion order.

- **Opt-in source provider parallel collection.** Known parallel-safe source
  providers can run through an explicit thread-pool path. Unsafe providers remain
  serial barriers and are still collected; they are not skipped.

- **Scout chunk join contract.** Scout runtime instructions now treat child
  chunk outputs as scratch material. The parent must deterministically join chunk
  outputs before writing final workflow artifacts. Default topology may write
  both `candidate_claims.json` and `screened_candidates.json`; strict topology
  keeps `screened_candidates.json` owned by Screener.

- **Quality gate evaluation helper.** Gate finding evaluation is now isolated in
  a read-only helper with helper-level opt-in parallel execution. The normal
  `gates check` command still owns report writing, legacy projection updates,
  and event emission as one serial transaction.

- **Stage runtime/model provenance.** `state stage-complete` and
  `state finalize-complete` can record explicit runtime/model values in workflow
  state and event log metadata. This is audit provenance only.

- **Owner-stage repair transaction.** `repair start` and `repair complete` give
  the Orchestrator a deterministic owner-stage repair path instead of guessing
  which frozen artifact to patch. Repair state is recorded in workflow state and
  event log, and contaminated runs remain non-reference-eligible.

- **Onboarding DOCX title fix.** Onboarding brief titles and DOCX heading
  configuration are kept separate.

## Evidence

- Full deterministic test suite runs in CI without LLM calls.
- CI includes generated-agent parity, runtime asset parity, version
  consistency, capability registration, release consistency, public-safety
  checks, source-provider coverage, and package-install smoke tests.
- Regression coverage includes source join ordering, mixed safe/unsafe provider
  collection, Scout chunk handoff contracts, helper-level gate serial/parallel
  equivalence, runtime/model provenance recording, and owner-stage repair
  hardening.

## What This Release Does Not Claim

MABW v0.8.4 does not claim faster end-to-end runtime, output-quality
improvement, semantic support verification, model-performance measurement, or
Codex parity.

`multi-agent-brief gates check` remains serial by default in the user-facing
CLI. Parallel gate evaluation is currently helper-level opt-in infrastructure.

Scout chunk parallelism is a runtime contract, not a Python Scout executor.
Chunk worker outputs are scratch/intermediate runtime material; only the joined
final artifacts count for stage completion.

Stage runtime/model provenance is recorded only when completion commands are
called with explicit `--runtime` / `--model` values. Normal runtime handoffs do
not automatically supply those flags yet.

## Get Started

Install from source:

```bash
git clone https://github.com/Stahl-G/multi-agent-brief-workflow.git
cd multi-agent-brief-workflow
bash scripts/setup.sh
```

Create a real brief workspace:

```bash
multi-agent-brief onboard
multi-agent-brief init ~/mabw-workspace --from-onboarding onboarding.json
multi-agent-brief run --workspace ~/mabw-workspace
```

For explicit stage provenance during manual/runtime-contract debugging:

```bash
multi-agent-brief state stage-complete \
  --workspace ~/mabw-workspace \
  --stage scout \
  --reason "Scout completed." \
  --runtime hermes \
  --model claude-sonnet-4
```
