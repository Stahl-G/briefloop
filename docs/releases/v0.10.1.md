# BriefLoop v0.10.1 — Product OS PolicyProfile Layer

v0.10.1 adds the experimental Product OS PolicyProfile layer on top of the
existing BriefLoop control spine. It introduces product-layer report contracts,
packaged defaults, resolved policy projection, a limited deterministic gate
adapter, and public-safe dogfood fixtures.

This release does not add industry compliance judgment, investment-advice
detection, internet rumor verification, IR/disclosure readiness, semantic truth
proof, automatic publication, or release authority.

## What Changed

- Added experimental `ReportSpec` and `ReportPack` contracts plus packaged
  `market_weekly` and `management_monthly` packs.
- Added `briefloop` as a shell CLI alias for `multi-agent-brief`, and
  `/briefloop` as the Claude writer command alias for the existing five-verb
  `/mabw` surface.
- Added `briefloop new <pack> <workspace>` /
  `multi-agent-brief new <pack> <workspace>` for conservative local-first
  workspace skeletons from packaged ReportPacks.
- Added experimental ReportTemplate registry and delivery/audit bundle manifest
  projection over already finalized workspace artifacts.
- Added experimental PolicyProfile registry with packaged
  `manufacturing_default`, `finance_default`, and `internet_default` defaults.
- Added resolved PolicyProfile visibility in `validate-report-spec`,
  read-only status, and generated runtime handoff artifacts.
- Added a limited PolicyProfile deterministic gate adapter that can tighten
  existing quality-gate strictness and reader-final forbidden-phrase checks.
- Added public-safe synthetic PolicyProfile dogfood fixtures for resolved
  projection, deterministic gate-adapter strictness, and reader-final
  forbidden-phrase checks.

## Compatibility

This release keeps these compatibility surfaces:

- `multi-agent-brief` remains the stable engine CLI.
- `briefloop` is a shell CLI alias.
- `/briefloop` is the Claude writer command.
- `/mabw` remains a compatibility Claude command.
- `multi_agent_brief` remains the Python package/module path.
- `multi-agent-brief-workflow` remains the Python distribution package name.
- Existing workspace artifacts, MABW experiment IDs, and control files are not
  renamed.

## Experimental Status

ReportSpec, ReportPack, ReportTemplate, and PolicyProfile support remains
experimental. These surfaces describe product metadata and deterministic
defaults over the existing control spine. They do not run subagents, render
templates, bypass gates, deliver reports, or authorize publication.

PolicyProfiles may tighten existing deterministic checks; they do not create a
second gate engine or become a compliance, truth, investment, rumor, or
publication authority.

## Boundaries

BriefLoop still does not prove truth, eliminate hallucinations, replace human
review, provide investment advice, or make reports automatically ready to send.

v0.10.1 does not add a lite mode, force-deliver path, hidden auto-publication,
IR/disclosure readiness, semantic support scoring, support-sufficiency gates,
human adjudication queues, or release eligibility.
