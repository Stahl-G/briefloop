# MABW v0.8.1 — Fast Rerun And Experiment Registration

MABW v0.8.1 is a measurement-infrastructure release. It adds hash-verified
fast-rerun foundations and the first MABW-080 experiment registration command,
while keeping the core claim narrow: **traceability and controlled comparison
surfaces, not semantic proof or output-quality scoring yet.**

## What's New

- **Control-trace timing projection.** `status` and archived run manifests now
  expose event-log-derived timing buckets. Timing is read-only, comes from the
  control trace, and reports unknown, incomplete, or contaminated states without
  guessing exact model runtime.

- **Frozen fact-layer archive.** Finalized run archives now include a
  fact-layer projection covering durable source evidence, input classification,
  candidate claims, screened candidates, and the Claim Ledger. `source_candidates`
  remains planning-only and cannot satisfy the fact layer.

- **Fast-rerun import transaction.** `multi-agent-brief state import-fact-layer`
  can import a complete archived fact layer into a fresh workspace for
  same-evidence downstream reruns. It copies bytes, verifies hashes, records the
  import in runtime state, and marks upstream fact-layer stages as satisfied by
  import without fabricating stage-complete history.

- **Fast-rerun handoff and freshness checks.** `multi-agent-brief run --recipe
  fast-rerun` requires a valid imported fact layer, starts at Analyst, and does
  not rerun source-discovery, Scout, Screener, or Claim Ledger. Finalize-complete
  rechecks imported source freshness against the target workspace before archive.

- **MABW-080 run registration.** `multi-agent-brief experiments 080
  register-run` writes one `run_record.json` for a completed workspace run and
  an existing 080 case. Registration validates archive bytes and records whether
  the run's frozen fact layer matches the case fact layer.

## Evidence

- Full deterministic test suite: 1400+ tests, no LLM calls.
- Release checks include generated-agent parity, version consistency,
  capability registration, public-safety scan, and release consistency.
- Public-safe fast-rerun fixtures cover clean fact-layer import, no delivery
  output after import, and rejection of source-plan-only archives.

## What This Release Does Not Claim

MABW v0.8.1 does not claim output-quality improvement, autonomous repair,
semantic proof, strict source-to-claim truth, Codex parity, or completed
MABW-080 scoring.

Fast-rerun is Experimental. It is not a gate-skipping lite mode: writer,
auditor, gates, finalize-complete, human delivery, and archive still remain in
the downstream loop.

MABW-080 is Experimental. `register-run` records experiment metadata only; it
does not score guidance manifestation, summarize experiments, scaffold
conditions, or prove that a brief improved.

## Get Started

Install from source:

```bash
git clone https://github.com/Stahl-G/briefloop.git
cd briefloop
bash scripts/setup.sh
```

Create a real brief workspace:

```bash
multi-agent-brief onboard
multi-agent-brief init ~/mabw-workspace --from-onboarding onboarding.json
multi-agent-brief run --workspace ~/mabw-workspace
```

Fast-rerun and MABW-080 commands are experimental control/measurement tools.
For normal briefing work, start with onboarding and the standard runtime handoff.
