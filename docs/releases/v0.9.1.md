# BriefLoop v0.9.1 — Atomic Claim Graph Foundation

v0.9.1 adds the first experimental Atomic Claim Graph control surfaces. The
graph is an optional structured decomposition of frozen Claim Ledger claims. It
is not source evidence, semantic proof, or a support-sufficiency score.

## What Changed

- Added optional `output/intermediate/atomic_claim_graph.json` schema and
  runtime validation.
- Added deterministic whole-ledger coverage and Claim Ledger type-consistency
  checks when the graph is present.
- Updated Analyst and Editor contracts so they may read a valid graph as an
  optional decomposition aid, but must not create, edit, repair, extend, or cite
  it.
- Added non-blocking quality-gate projection metadata for atom ID residue and
  Claim Ledger citation coverage.
- Extended reader-final residue checks so atom IDs and Atomic Claim Graph
  process wording cannot leak into delivery output.

## Compatibility

This release does not rename:

- `multi-agent-brief` CLI
- `/mabw` Claude commands
- `multi_agent_brief` Python package/module path
- `multi-agent-brief-workflow` distribution package name
- workspace formats
- artifact names
- MABW experiment IDs such as `MABW-080`

## Boundaries

BriefLoop still does not prove truth, eliminate hallucinations, replace human
review, or make reports automatically ready to send.

v0.9.1 does not implement Evidence Span Registry, Claim-Support Matrix,
semantic support scoring, support-sufficiency gating, or automatic
atomization. It only adds an experimental graph contract and deterministic
visibility surfaces that later support-sufficiency work can consume.
