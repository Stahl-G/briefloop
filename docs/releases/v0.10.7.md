# BriefLoop v0.10.7 — Product OS Hardening And Quality Panel

v0.10.7 ships the accumulated experimental Product OS hardening line on top of
the existing BriefLoop control spine. It adds product-layer report contracts,
source evidence materialization, cleaner delivery/audit bundle surfaces,
internal approval records, and Quality Panel projections.

This release does not add quality scoring, semantic truth proof, delivery
approval, public release authority, industry compliance judgment,
investment-advice detection, internet rumor verification, or IR/disclosure
readiness.

## What Changed

- Added and hardened experimental ReportSpec, ReportPack, ReportTemplate, and
  PolicyProfile product-layer contracts.
- Added conservative workspace skeleton creation, user-facing pack aliases, and
  deterministic PolicyProfile resolution for zero-config workspaces.
- Added the experimental `solar_industry_periodic` ReportPack and
  `solar_manufacturing_default` PolicyProfile as public-safe product dogfood
  contracts without customer-specific defaults.
- Added read-only ReportTemplate section-order, section-conformance, and
  render-plan projections in status and runtime handoff artifacts.
- Added bundle projection hygiene, deterministic clean `delivery_bundle.zip`
  and `audit_bundle.zip` archives, DOCX hyperlink rendering for source URLs,
  and audit-bundle inclusion for valid Quality Panel artifacts.
- Added experimental durable source evidence pack materialization and source
  taxonomy normalization for provider, retrieval, reader-facing category, and
  underlying evidence type.
- Added experimental Evidence Extract source/scope registration and SourceHub
  Lite setup commands for local files, RSS feeds, and runtime web-search
  handoff tasks.
- Added Source Appendix and audit-trace hardening for source metadata,
  retrieval taxonomy, source hashes, and metadata completeness warnings.
- Added deterministic discard audit validation for object-shaped screened
  candidate artifacts.
- Added experimental internal release-mode approval records with current-run
  and event-log linkage validation.
- Added experimental Quality Panel JSON, Markdown, and static HTML projections
  plus `briefloop quality summarize`.

## Compatibility

This release keeps these compatibility surfaces:

- `multi-agent-brief` remains the stable engine CLI.
- `briefloop` remains a shell CLI alias.
- `/briefloop` remains the Claude writer command.
- `/mabw` remains a compatibility Claude command.
- `multi_agent_brief` remains the Python package/module path.
- `multi-agent-brief-workflow` remains the Python distribution package name.
- Existing workspace artifacts, MABW experiment IDs, and control files are not
  renamed.

## Experimental Status

ReportSpec, ReportPack, ReportTemplate, PolicyProfile, Evidence Extract,
SourceHub Lite, source evidence pack manifests, internal approval records, and
Quality Panel projections remain experimental Product OS surfaces.

They describe setup, metadata, source materialization, bundle packaging, or
read-only control summaries over the existing control spine. They do not run
subagents, prove source support, replace gates, repair artifacts, approve
delivery, publish reports, or authorize public release.

## Boundaries

BriefLoop still does not prove truth, eliminate hallucinations, replace human
review, provide investment advice, or make reports automatically ready to send.

v0.10.7 does not add a lite mode, force-deliver path, hidden
auto-publication, IR/disclosure readiness, semantic support scoring,
support-sufficiency gates, human adjudication queues, release eligibility,
external publication authorization, or a quality score.
