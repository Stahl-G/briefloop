# BriefLoop v0.9.0 Preview — Brand Rename And Public Framing

v0.9.0 establishes **BriefLoop** as the project name. Retired labels are not alternate product names and remain visible only where a literal compatibility identifier requires them.

## What Changed

- README and public documentation now use BriefLoop as the public project name.
- The public GitHub repository URL is `https://github.com/Stahl-G/briefloop`.
- Legacy CLI, command, module, schema, and archived experiment identifiers remain compatible without acting as product labels.
- `docs/briefloop-naming.md` defines the naming and compatibility policy.
- The [archived brief-loop engineering note](../archive/brief-loop-engineering.md)
  is historical design material only.
- Current engineering and support truth is defined by the
  [charter](../charter/README.md), [architecture status](../architecture-status.md),
  and [support matrix](../support-matrix.md).
- The public v0.9 roadmap now points toward support sufficiency and
  brief-loop engineering instead of distribution/reference workflows.

## Compatibility

This preview does not rename:

- `multi-agent-brief` CLI
- `/mabw` Claude commands
- `multi_agent_brief` Python package/module path
- `multi-agent-brief-workflow` distribution package name
- workspace formats
- artifact names
- reference-run paths
- frozen experiment IDs such as `MABW-080`

## v0.9 Direction

The minimum v0.9 support-sufficiency path is:

```text
Atomic Claim Graph -> Evidence Span Registry -> Claim-Support Matrix
```

Later v0.9.x candidates include semantic assessment proposals, human
adjudication, coverage/omission gates, semantic regression, release
eligibility, quality packs, and finding-to-repair workflows.

## Public-Safe 090 Pilot Note

The repo now documents one public-safe synthetic BriefLoop-090
`auditable_brief` pilot:

- [BriefLoop-090 A-controlled auditable-brief pilot](../reference-runs/briefloop-090-a-controlled-pilot.md)

The pilot used the shipped, frozen `MABW-080` experiment command namespace with
condition-blind, hash-bound assessment. In that single synthetic case, the
memory condition showed the approved guidance without obvious harm, while
prompt-only over-applied the same guidance.

This is a documented single-case observation, not a general output-quality
claim and not a management-ready delivery claim.

## Boundaries

BriefLoop does not prove truth, eliminate hallucinations, replace human review,
or make reports automatically ready to send.

v0.9.0 is a brand/public-framing preview. It does not implement Atomic Claim
Graph, Evidence Span Registry, Claim-Support Matrix, or formal A-controlled
output-quality claims.
